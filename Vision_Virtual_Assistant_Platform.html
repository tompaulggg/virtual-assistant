Die Idee

Eine SaaS-Plattform fur AI-Assistenten — jeweils spezialisiert auf eine Berufsgruppe. Jeder Assistent hat eine eigene Personlichkeit, kennt seinen Nutzer und wird mit jeder Interaktion schlauer. Gemeinsamer technischer Kern, vertikale Spezialisierung.

Architektur

Telegram ─── Nutzer schreibt Nachricht │ ▼ Bot (Authentifizierung, Rate Limiting) │ ▼ Brain (Claude API) ─── versteht Intent, wahlt Aktion │ ├──▶ Actions (Ghostwriter, Todos, E-Mail, Kalender...) │ ├──▶ Auto-Learning (extrahiert Fakten aus jedem Gesprach) │ ├──▶ Sub-Agents (Code ausfuhren, recherchieren, Dateien erstellen) │ └──▶ Scheduler (Briefings, Erinnerungen, proaktive Nachrichten) │ ▼ Supabase ─── Langzeitgedachtnis (Wissen, Kontakte, Projekte)

Zukunftsvision: Susi als Orchestrator

Der Nutzer gibt einen Auftrag. Susi erkennt was zu tun ist und delegiert an spezialisierte Sub-Agents:

"Erstell eine Prasentation uber AI in der Zahnmedizin" Susi (Brain) │ ├──▶ Research-Agent → recherchiert das Thema ├──▶ Design-Agent → erstellt HTML/PDF └──▶ File-Agent → schickt Datei per Telegram Ergebnis: Fertige Prasentation als Datei im Chat

Business Model

Strategie

Phase 1: Piloten validieren (Lena fur Esther, Susi fur Thomas) — kostenlos, lernen was funktioniert.

Phase 2: Arzt-Vertical als erster zahlender Kunde. Preis: ~149-299/Monat pro Praxis.

Phase 3: Weitere Verticals. Jeder neue Vertical = gleicher Core, neue Actions + Personlichkeit.

Skalierung: Ein neuer Vertical kostet Wochen statt Monate, weil der Core steht.
